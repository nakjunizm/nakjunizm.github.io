haproxy & keepalived лЎң н”„лЎқмӢң кө¬м¶•н•ҳкё°

2020-02-20
haproxy, keepalived

overview

VM л‘җлҢҖ(мқҙмғҒ)м—җ haproxyлҘј м„Өм№ҳн•ҳм—¬ L4(лҳҗлҠ” L7) switch мһҘл№„лҘј лҢҖмІҙ н•  мҲҳ мһҲлҸ„лЎқ н•ҳлҠ”кІҢ лӘ©н‘ң мһ…лӢҲлӢӨ. haproxyк°„мқҳ HA(High availability)лҠ” keepalivedлҘј мӮ¬мҡ©н•ҳм—¬ VIPлҘј л„ҳкё°лҠ” мӢқмңјлЎң кө¬м„ұ н•©лӢҲлӢӨ.
мӮ¬мӢӨ м§ҖкёҲ L7мқ„ мӮ¬мҡ©н•ҳкі  мһҲлҠ”лҚ° мқҙл ҮкІҢ н•ҳл ӨлҠ” мқҙмң лҠ” лӢӨмқҢкіј к°ҷмҠөлӢҲлӢӨ.

  1. LoadBalancerлҘј мӮ¬мҡ©н•  мҲҳлҠ” мһҲлҠ” нҷҳкІҪ мқҙм§Җл§Ң лӮҙк°Җ м»ЁнҠёлЎӨ н•  мҲҳ м—ҶлҠ” мғҒнҷ© (л„ӨнҠёмӣҢнҒ¬ л¶Җм„ңм—җм„ң лӢҙлӢҙ).
  2. кІҖмҰқкі„м—җм„ң мқҙкІғм ҖкІғ кө¬м„ұн•ҳл Өкі  н•ҙлҸ„ L7 кҙҖл Ё м„Өм •мқҖ SR(Service Request) нӢ°мј“мқ„ л§Ңл“Өм–ҙм•ј н•ҳкі , мөңмҶҢ мқјмЈјмқј м „м—җлҠ” SRмқ„ мҳ¬л Өм•ј лӮҙк°Җ мӣҗн•ҳлҠ” кё°к°„м—җ л§һм¶°м„ң м„Өм •мқ„ л°”кҝҖ мҲҳ мһҲмқҢ.

кө¬м„ұн•  лӘЁмҠө(TO-BE)

нҸүмғҒмӢңм—җлҠ” мң„ к·ёлҰјкіј к°ҷмқҙ stg1м—җ VIP 12.34.56.129 мқҙ л¶ҷм–ҙ мһҲкі  лҰ¬нҖҳмҠӨнҠёк°Җ л“Өм–ҙмҳӨл©ҙ stg1мқҳ haproxyлҘј нҶөн•ҙ stg1кіј stg2лЎң л°ёлҹ°мӢұ лҗ©лӢҲлӢӨ.
к·ёлҹ¬лӢӨк°Җ stg1мқҙ лӢӨмҡҙлҗҳл©ҙ м•„лһҳ к·ёлҰјкіј к°ҷмқҙ VIPк°Җ stg2лЎң мҳ®кІЁ к°ҖкІҢ лҗҳкі , stg2 -> stg2лЎңл§Ң лҰ¬нҖҳмҠӨнҠёлҘј ліҙлӮҙлӢӨк°Җ stg1мқҙ лӢӨмӢң мӮҙм•„ лӮҳл©ҙ stg1лЎңлҸ„ лҰ¬нҖҳмҠӨнҠёлҘј лӮҳлҲ  мӨҚлӢҲлӢӨ.

н…ҢмҠӨнҠё лӘ©м Ғмқҳ кІҖмҰқкі„мқҳ кө¬м„ұмқҙлқјм„ң мқҙл ҮкІҢ м—ҙм•…н•ҳкІҢ кө¬м„ұ лҗҳм—Ҳм§Җл§Ң, haproxy л‘җлҢҖлҘј лі„лҸ„лЎң кө¬м„ұн•ҳкі  л°‘м—җ лЎңл“ңл°ёлҹ°мӢұ лҗҳлҠ” м„ңлІ„л“Өмқ„ м—¬лҹ¬лҢҖ л¶ҷмқҙлҠ”кІҢ мқјл°ҳм Ғмқё кө¬м„ұмһ…лӢҲлӢӨ.

мӨҖл№„мӮ¬н•ӯ

haproxyмҷҖ keepalivedлҘј м„Өм№ҳн•  VM 2лҢҖ (yum repository мӮ¬мҡ© к°ҖлҠҘн•ң)
floating ipлЎң мӮ¬мҡ©н•  VIP 1к°ң

haproxy м„Өм№ҳ

haproxy.cfg мҲҳм •

balance option(algorithm) м—җ лҢҖн•ҙ к°„лһөн•ҳкІҢ м •лҰ¬ н–ҲмҠөлӢҲлӢӨ. (мһҗм„ён•ң лӮҙмҡ©мқҖ https://cbonte.github.io/haproxy-dconv/1.7/configuration.html#4-balance м°ёмЎ°)

  • roundrobin: мҡ°лҰ¬к°Җ мһҳ м•Ңкі мһҲлҠ” вҖҳк·ёвҖҷ roundrobin л°©мӢқкіј лҸҷмқј. мҲңм„ңлҢҖлЎң лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • static-rr: weight м—җ л”°лқј л¶„л°°н•ҙ мЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ. weightк°Җ лҶ’мқҖ м„ңлІ„лЎң лҚ” л§ҺмқҖ лҰ¬нҖҳмҠӨнҠёлҘј лҸҢл ӨмӨҚлӢҲлӢӨ.
  • leastconn: connectionмқҙ м ңмқј м ҒмқҖ м„ңлІ„м—җ лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • first: м„ңлІ„ id мҲңм„ңлҢҖлЎң лҸҢл ӨмЈјлҠ”лҚ°, м•һм—җ м„ңлІ„к°Җ м°Ёл©ҙ к·ё лӢӨмқҢ м„ңлІ„лЎң лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • soucre: IP hash к°’мқ„ м„ңлІ„ лҢ“мҲҳлЎң лӮҳлҲҲ к°’м—җ л”°лқј лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • uri: uri hashк°’мқ„ мҙқ м„ңлІ„ лҢ“мҲҳлЎң лӮҳлҲҲ к°’м—җ л”°лқј лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • url_param: urlмқҳ мҝјлҰ¬мҠӨнҠёл§Ғмқҳ hash к°’мқ„ мҙқ м„ңлІ„ лҢ“мҲҳлЎң лӮҳлҲҲ к°’м—җ л”°лқј лҸҢл ӨмЈјлҠ” л°©мӢқ мһ…лӢҲлӢӨ.
  • hdr(): http headerмқҳ name к°’мқ„ ліҙкі  лҸҢл ӨмЈјлҠ”лҚ° nameмқҙ headerм—җ м—Ҷмңјл©ҙ rrлЎң лҸҢл Ө мӨҚлӢҲлӢӨ..
  • rdp-cookie(): TCP лҰ¬нҖҳмҠӨнҠёмқҳ hash к°’мқ„ ліҙкі  лҸҢл ӨмӨҢ. equivalent ACL вҖҳreqrdpcookie()вҖҷ function. cookieк°Җ м—Ҷмңјл©ҙ rrлЎң лҸҢл Ө мӨҚлӢҲлӢӨ.

haproxy logging м„Өм •

кё°ліём ҒмңјлЎң haproxy м„Өм •мқ„ ліҙл©ҙ м•„лһҳмҷҖ к°ҷмқҙ logлҘј 127.0.0.1 лЎң ліҙлӮҙкі  мһҲлҠ”кІғмқ„ нҷ•мқё н•  мҲҳ мһҲмҠөлӢҲлӢӨ.
мқҙл ҮкІҢ ліҙлӮҙл©ҙ rsyslogлЎң ліҙлӮҙлҠ” кІғ мқҙм§Җл§Ң, rsyslogм—җлҠ” кҙҖл Ён•ҙм„ң м•„л¬ҙлҹ° м„Өм •мқ„ н•ҳм§Җ м•Ҡм•ҳкё° л•Ңл¬ём—җ лЎңк№…мқҙ лӢ№м—°нһҲ м•Ҳлҗ©лӢҲлӢӨ.

/etc/rsyslog.conf мҲҳм •

haproxyлҠ” лЎңк·ёлҘј UDPлЎң ліҙлӮҙмЈјкё° л•Ңл¬ём—җ rsyslogм—җм„ң udpлҘј мІҳлҰ¬ н•  мҲҳ мһҲлҸ„лЎқ м•„лһҳ л‘җ к°Җм§Җ мҳөм…ҳмқҳ мЈјм„қмқ„ н•ҙм ң н•©лӢҲлӢӨ

/etc/rsyslog.d/haproxy.conf нҢҢмқј мғқм„ұ

rsyslogм—җ udp лҘј л°ӣмқ„ мҲҳ мһҲкІҢ мІҳлҰ¬лҘј н•ҙ мӨ¬мңјлӢҲ мӢӨм ңлЎң haproxy лЎңк·ём—җ лҢҖн•ң м„Өм • нҢҢмқјмқ„ л§Ңл“Өм–ҙ мӨҚлӢҲлӢӨ.

м¶”к°ҖлЎң /nakjunizm/data/logs/util/haproxy л””л үнҶ лҰ¬ м—Ҷмңјл©ҙ лҜёлҰ¬ мғқм„ұ. (mkdir -p /nakjunizm/data/logs/util/haproxy)

rsyslog мһ¬мӢңмһ‘

sysctl м„Өм •

keepalivedлҘј мқҙмҡ©н•ҙм„ң floating ipлҘј мҳ®кІЁлӢӨлӢҲкІҢ н•ҳкё° мң„н•ҙ м•„лһҳ л‘җк°Җм§Җ м„Өм •мқҙ н•„мҡ” н•©лӢҲлӢӨ.
net.ipv4.ip_forward лҠ” NIC к°„мқҳ forwarding мқ„ н—Ҳмҡ© н•ҙ мЈјлҠ” мҳөм…ҳмңјлЎң, мҳҲлҘјл“Өм–ҙ eth0мңјлЎң л“Өм–ҙмҳЁ лҰ¬нҖҳмҠӨнҠёлҘј eth1лЎң нҸ¬мӣҢл”© мӢңнӮ¬л•Ң н•„мҡ” н•©лӢҲлӢӨ.
eth0м—җ VIPлҘј м¶”к°ҖлЎң bind мӢңнӮӨлҠ” кІҪмҡ°м—җлҠ” мң„ мҳөм…ҳмқ„ 1лЎң мҲҳм •н•  н•„мҡ”к°Җ м—Ҷкі , лӢӨлҘё NICм—җ VIPлҘј л¶ҷмқҙлҠ” кІҪмҡ°м—” н•„мҡ”н•ң кІғмңјлЎң мғқк°Ғ лҗ©лӢҲлӢӨ.
net.ipv4.ip_nonlocal_bind лҠ” network interfaceм—җ н• лӢ№лҗҳм§Җ м•ҠмқҖ IPлҘј bind н• мҲҳ мһҲкІҢ allow н•ҳлҠ” мҳөм…ҳмңјлЎң, мҡ°лҰ¬лҠ” VIPлҘј bind н• кІғмқҙкё° л•Ңл¬ём—җ 1лЎң м„Өм •н•ҳм—¬ enable мӢңнӮ¬ кІғ мһ…лӢҲлӢӨ.
